# Contributing

## Code quality

When creating a pull request (merge request) please format your code with the provided `.clang-format`. 

## Review workflow

Once you create a pull request (merge request) the code must be reviewed by at least one developer. 

Here are the review rules:
- Once a comment is placed by a reviewer, the reviewer is the only one that can resolve the comment.
- Once a comment is addressed by the author, author must react to the comment either with a reaction or a followup message that the comment was addressed.
